
一、上电与启动
━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1 设备通电后无任何反应，指示灯全灭
  可能原因：电源线未接好 / 外部供电异常 / 内部电源模块保护
  排查步骤：
    ① 确认现场供电插座是否有电（可用万用表测100-240V输入端）
    ② 检查电源线是否插紧，接头是否拧到位
    ③ 若供电正常仍不亮，断开电源等待30秒后重新上电
    ④ 仍无效则可能是内部电源模块故障，返厂检修

1.2 通电后电源灯亮（绿灯），运行灯不亮
  可能原因：系统未完成启动 / 内核崩溃 / 固件损坏
  排查步骤：
    ① 等待2-3分钟，首次启动需要初始化系统服务
    ② 若超过5分钟仍未亮起，插上网线查看网络指示灯判断系统是否正常启动
    ③ 尝试长按复位键（RST）5秒强制重启
    ④ 若多次重启无效，需通过SD卡重刷固件

1.3 设备频繁自动重启
  可能原因：供电电压不稳定 / 散热不良触发过热保护 / 系统负载过高
  排查步骤：
    ① 检查现场电压是否在100-240V范围内，波动过大需先解决供电问题
    ② 触摸外壳是否过热，附近有热源
    ③ 拔掉外接USB设备，排除短路或过载
    ④ 如近期做过OTA升级，可能是固件问题，联系技术支持回滚版本
     ⑤ 硬件故障

二、网络通信
━━━━━━━━━━━━━━━━━━━━━━━━━━━

2.1 网口指示灯不亮，无法上网
  可能原因：网线松动 / 网线损坏 / 交换机端口未开启 / 网口硬件故障
  排查步骤：
    ① 重新插拔网线，确认听到卡扣声
    ② 更换一根已知好的网线测试
    ③ 换交换机上的另一个端口试试
    ④ 用笔记本电脑插同一根网线，确认网络本身正常
    ⑤ 网口灯仍不亮，可能是PHY芯片损坏，走售后更换

2.2 能连局域网但无法访问外网
  可能原因：DNS配置错误 / 网关地址不对 / 路由策略问题
  排查步骤：
    ① 检查DNS是否为114.114.114.114或8.8.8.8
    ② 确认默认网关地址是否正确
    ③ 用ping测试：ping 114.114.114.114（IP通说明路由正常）
    ④ 若IP通域名不通，检查DNS设置

2.3 WiFi搜索不到或连接不稳定
  可能原因：天线未接 / 信号干扰 / 信道不兼容
  排查步骤：
    ① 确认WiFi天线已固定，且无断裂
    ② 天线安装位置不对
    ③ 用手机在同一位置测试WiFi信号强度

2.4 4G/5G模块无法识别
  可能原因：SIM卡未插好 / SIM卡欠费 / 模块松动
  排查步骤：
    ① 断电，检查SIM卡槽方向是否正确，金属触点是否清洁
    ② 将SIM卡插入手机测试是否能上网
    ③ 重启设备，等待30秒后在Web管理页查看模块状态
    ④ 仍未识别，打开外壳检查模块排线是否松动


三、RS-485及I/O接口
━━━━━━━━━━━━━━━━━━━━━━━━━━━

3.1 485通信乱码或丢包
  可能原因：波特率不匹配 / 接地问题 / 线缆过长 / 终端电阻缺失
  排查步骤：
    ① 确认上下位机波特率、数据位、停止位、校验位完全一致
    ② 检查485屏蔽层单端接地，不要两端都接地
    ③ 通信距离超过500米需降低波特率（建议9600以下）
    ④ 总线两端各加一个120Ω终端电阻
    ⑤ USB转485直连PC查看通讯数据包

3.2 DI/DO信号不响应
  可能原因：接线错误 / 外部供电异常 / 端口烧毁
  排查步骤：
    ① DI连线是否正确连接，触电是否能形成正确通路
    ② DO为干节点输出，确认外部负载电流不超过额定值
    ③ 用万用表测量DI端口电平：开路应为高电平，短接应为低电平
    ④ DO可用万用表电阻档测量，导通时接近0Ω 或打开到通路检测端，通路提示声响
    ⑤ 若所有IO均无效，可能是PLC模块掉配置，需远程重新下发

3.3 4-20mA采集读数不准
  可能原因：信号源问题 / 线路干扰 / 采集通道配置错误
  排查步骤：
    ① 用信号发生器直接输入4mA和20mA标准信号，确认设备端读数
    ② 检查现场仪表是否正常工作，断开后单独测试设备通道
    ③ 确认线缆是否与电力线平行走线
    ④ 如读数跳变剧烈，加装信号隔离器


四、AI推理与摄像头
━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1 摄像头画面无法显示
  可能原因：IP地址冲突 / 协议不兼容 / 带宽不足
  排查步骤：
    ① 用电脑VLC/PotPlayer测试摄像头RTSP流是否正常
    ② 确认摄像头IP和AIEdgeBox在同一网段
    ③ Web管理页 → 算法配置 → 检查摄像头通道是否绑定正确
    ④ 如果接入多路1080P以上摄像头，确认千兆网口带宽足够

4.2 AI识别准确率明显下降
  可能原因：摄像头角度偏移 / 光线变化 / 模型待更新
  排查步骤：
    ① 检查摄像头是否被异物遮挡或有人为转动
    ② 同一场景用手机拍照对比实际画面与算法输入画面
    ③ 若现场光照条件发生变化（夜间/逆光），需调整摄像头参数
    ④ OTA推送最新的模型文件，覆盖旧版本

4.3 系统提示"GPU/算力不足"
  可能原因：同时运行的算法过多 / 有算法进程异常退出未释放资源
  排查步骤：
    ① 检查当前加载的算法数量，10.4T算力建议不超过3路复杂模型
    ② 登录Web管理 → 系统监控 → 查看CPU/内存/NPU占用率
    ③ 重启算法服务释放资源：Web管理页 → 算法管理 → 一键重启
    ④ 如频繁出现，需排查是否有算法进程内存泄漏


五、OTA升级
━━━━━━━━━━━━━━━━━━━━━━━━━━━

5.1 OTA升级失败，进度条卡住不动
  可能原因：网络不稳定 / 升级包下载不完整 / 存储空间不足
  排查步骤：
    ① 检查设备网络是否正常，ping升级服务器是否通
    ② 确认设备剩余存储空间大于升级包大小
    ③ Web管理页 → 系统 → 存储管理，清理不需要的日志文件

5.2 升级成功后设备反复重启
  可能原因：固件版本不兼容 / 升级过程断电导致分区损坏
  排查步骤：
    ① 长按复位键进入恢复模式
    ② 通过SD卡刷回上一个稳定版本
    ③ 报告给技术支持，提供设备SN号和故障时的日志

5.3 升级后部分功能异常
  可能原因：新固件配置项有变更
  排查步骤：
    ① 清除浏览器缓存，重新登录Web管理页
    ② 检查配置参数是否因版本升级被重置，重新配置
    ③ Web管理页 → 系统 → 恢复出厂设置（注意备份原有配置）


六、存储与日志
━━━━━━━━━━━━━━━━━━━━━━━━━━━

6.1 设备提示"磁盘空间不足"
  可能原因：日志文件积累 / 算法运行产生大量缓存 / SD卡损坏
  排查步骤：
    ① 登录Web管理 → 系统 → 存储管理，查看各分区占用情况
    ② 清理日志：设置日志保留天数
    ③ 检查磁盘或sd健康状态

6.2 系统时间不正确
  可能原因：电池耗尽 / NTP服务不可达 / 时区配置错误
  排查步骤：
    ① 设备断电超过24小时再上电，时间可能复位
    ② Web管理页 → 系统 → 时间设置，确认时区为UTC+8
    ③ 确保NTP服务器可达（默认 pool.ntp.org）
    ④ 若设备部署在内网无外网，手动设置时间或搭建内网NTP


七、环境与物理
━━━━━━━━━━━━━━━━━━━━━━━━━━━

7.1 运行中突然宕机并无法再次启动
  可能原因：高温触发硬件保护 / 进水短路
  排查步骤：
    ① 让设备冷却至室温后再尝试上电
    ② 检查外壳是否有水渍或凝露


7.2 UPS电池续航明显缩短
  可能原因：电池老化 / 充放次数过多 / 环境温度过低
  排查步骤：
    ① 宽温级别电池在低温下容量会显著衰减，属正常现象
    ② 电池循环充放超过500次后容量降至80%以下，建议更换
    ③ 需更换电池时请联系售后，勿自行拆解电池包

7.3 外壳密封不严，有进水风险
  可能原因：密封圈老化 / 安装时螺丝未对角拧紧 / 航空接头未拧到位
  排查步骤：
    ① 每6个月检查一次密封圈，发现有裂纹或变形及时更换
    ② 打开维护后重新合盖时，螺丝必须按对角线顺序分次拧紧
    ③ 所有未使用的航空接头需用配套防水堵头封闭
    ④ 防爆型号需特别注意接地线的完整性

7.4 电子锁无法正常开关
  可能原因：锁舌卡滞 / 电子模块供电异常
  排查步骤：
    ① 用手轻微推拉盖板再尝试开锁，释放锁舌应力
    ② 检查电子锁线缆是否被夹断
    ③ Web管理页检查电子锁状态反馈是否正常
